What is SIP?

Go back home

SIP (System Integrity Protection) is a security feature in certain versions of macOS that protects the most core and important files your Mac needs to operate. It stands for System Integrity Protection. Not all system files are protected by SIP (System Integrity Protection), just the really really really important ones. For example, your entire VM (Virtual Memory) disk has the full protection of the SIP (System Integrity Protection) because deleting Swapfiles or all-important Kernelcore file could tottaly wreck your system. What the SIP does is that you can't move, delete, duplicate, edit, or edit permissions on a file protected by SIP (System integrity Protection), not even when using the sudo, sudo rm, open; sudo rm, or cd; sudo rm commands, which normally would force any file to be deleted, edited, changed permissions, or moved, no matter how protected the file was. But SIP (System Integrity Protection) can block even that because SIP (System Integrity Protection) basically controls what you can and can't do with core system files. It is only applied to your startup disk, so SIP (System Integrity Protection) does not work in Recovery OS mode because Recovery OS Mode uses a different startup volume.

What SIP does

SIP does just what it sounds like: it protects the System's integrity. If you try to mess with core System directories, SIP will block you from deleting, modfying, moving, copying, and sometimes even opening them. But are all System files and folders protected by SIP? No. Only the most core of all that your computer would simply be tottally destroyed without. In fact, only /System (see top-level folders in Mac OS X), /private/var, apps that are pre-installed with Mac OS X, and all of the contents of /usr except /usr/local. /usr stands for Unix System Resources, but some people might just call it user.
