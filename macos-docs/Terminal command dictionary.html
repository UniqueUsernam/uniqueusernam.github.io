Go back home

sudo is not a command itself; rather a prefix for other commands, that adds one additional feature: any commnds in Terminal prefixed with sudo will run as if the root user, the highest level of power on a Mac, was executing it; it will be treated as a command from root user. So let's break this complicated command down: if you have a directory (or folder) called "Top-secret-stuff" located in "private", and you lock it, and when you try to get back in you can't, your first move is the Terminal. You run ls /private/top-secret-stuff, but it says permission denied. You add sudo to the beginning, enter your password at the prompt (the Terminal won't show what you type, just keep going), and guess what? It works! (Example: you have a locked folder called top-secret-stuff in the "private" folder, and you try this to get back in: ls /private/top-secret-stuff. Dosen't work. You try this: sudo ls /private/top-secret-stuff, enter your password, and poof! It works!) And when you add the -u option, that command (prefixed with sudo -u) will not be treated as a command from root user; if after the -u option and a space, you add a users name, it will be treated as a command from that user!
